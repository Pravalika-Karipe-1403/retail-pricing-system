Functional Requirement	                           Non-Functional Requirement
Describes WHAT system should do	                   Describes HOW WELL system should work

Functional
• Upload CSV pricing file
• Store pricing data in database
• View pricing records in table
• Filter by location, SKU, store
• Edit price
• Save updated price
• View price history

Non-Functional Requirements (Quality)

1. Performance

    System should load data within 2 seconds

2. Scalability

    System should support   

        • 3000 stores
        • Millions of records

3. Security

    • JWT Authentication
    • Role-based access

4. Availability

    System should be available 99.9%

5. Reliability

    No data loss during upload

6. Usability

    Easy UI with filters, pagination

7. Maintainability

    Code should be modular

8. Auditability

    Track price changes history
